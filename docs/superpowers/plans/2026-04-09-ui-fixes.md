## Task 1 — XmlGump: Reduce timer, snapshot coords, save on Dispose

**Files:**
- Modify: `src/ClassicUO.Client/Game/UI/XmlGumpHandler.cs` (class `XmlGump`, lines 805–932)

### Context

`XmlGump.OnMove` (line 893) sets `saveFileAfter = Time.Ticks + 10000`. If the game closes within 10 s of the last move, the save never fires. `SaveFile()` also reads `X`/`Y` on a background thread while the main thread can mutate them.

### Steps

- [ ] **Step 1: Reduce the save delay from 10 000 ms to 2 000 ms**

  In `XmlGumpHandler.cs`, `OnMove` method (~line 897):
  ```csharp
  // Before:
  saveFileAfter = Time.Ticks + 10000;
  // After:
  saveFileAfter = Time.Ticks + 2000;
  ```

- [ ] **Step 2: Snapshot X/Y before the background task in `SaveFile()`**

  In `SaveFile()` (~line 903), capture coords at the top before `Task.Run`:
  ```csharp
  private void SaveFile()
  {
      if (!savingFile)
      {
          savingFile = true;
          int snapshotX = X;
          int snapshotY = Y;

          if (!string.IsNullOrEmpty(FilePath) && File.Exists(FilePath))
          {
              XmlDocument xmlDoc = new XmlDocument();
              try
              {
                  xmlDoc.LoadXml(File.ReadAllText(FilePath));

                  if (xmlDoc.DocumentElement != null)
                  {
                      XmlElement root = xmlDoc.DocumentElement;
                      root.SetAttribute("x", snapshotX.ToString());
                      root.SetAttribute("y", snapshotY.ToString());

                      xmlDoc.Save(FilePath);
                  }
              }
              catch (Exception e) { GameActions.Print(e.Message); }
          }

          savingFile = false;
      }
  }
  ```
  Note: `SaveFile()` is now called both from `Task.Run` (periodic, async) and directly from `Dispose` (synchronous). The snapshot approach makes both safe.

- [ ] **Step 3: Add `Dispose()` override that saves synchronously before base dispose**

  Add after `SaveFile()` in `XmlGump` class (~line 931):
  ```csharp
  public override void Dispose()
  {
      // Save position synchronously before base.Dispose() clears state.
      // Only save if there are pending unsaved changes (saveFileAfter was set).
      if (SavePosition && saveFileAfter != uint.MaxValue)
      {
          SaveFile();
      }
      base.Dispose();
  }
  ```

## Task 3 — ScrollArea: Forward right-strip mouse clicks to scrollbar

**Files:**
- Modify: `src/ClassicUO.Client/Game/UI/Controls/ScrollArea.cs`

### Context

`ScrollArea` has a `private readonly ScrollBarBase _scrollBar`. When the user clicks 4px to the left of the scrollbar widget, the click lands on `ScrollArea` itself instead of the scrollbar child. Add an `OnMouseDown` override to forward those clicks.

`_scrollBar.X` is set to `Width - 14` (normal scrollbar) or `Width - 19` (ScrollFlag). Clicks at `x >= _scrollBar.X - 4` are treated as scrollbar clicks.

`Control.InvokeMouseDown(Point position, MouseButtonType button)` internally computes `localX = position.X - control.X - control.ParentX`. Pass the raw ScrollArea-local coords so `InvokeMouseDown` does the offset math correctly.

### Steps

- [ ] **Step 1: Add the `OnMouseDown` override**

  In `ScrollArea.cs`, add after the `OnMouseWheel` method (~line 187). The `_scrollBar` field is accessible since this override is within the same class:
  ```csharp
  protected override void OnMouseDown(int x, int y, MouseButtonType button)
  {
      if (button == MouseButtonType.Left && _scrollBar.IsVisible && x >= _scrollBar.X - 4)
      {
          // Pass raw ScrollArea-local coords; InvokeMouseDown subtracts _scrollBar.X/Y internally
          _scrollBar.InvokeMouseDown(new Point(x, y), button);
          return;
      }
      base.OnMouseDown(x, y, button);
  }
  ```

  Expected: found with `Point` parameter.

## Task 4 — BorderControl: Widen move hit area to 8 px

**Files:**
- Modify: `src/ClassicUO.Client/Game/UI/Gumps/WorldViewportGump.cs` (class `BorderControl`, line 382)

### Context

`BorderControl` currently has no `Contains` override, so the default `Control.Contains` returns `true` for the entire bounding box. The visual border is only 4 px thick. Adding an override restricts hit-testing to 8 px strips around the edges, making the drag handle twice as wide without changing the visual.

Content controls (buttons, labels) added after `BorderControl` win by z-order over the content area — this override only affects the border strips.

### Steps

- [ ] **Step 1: Add the `Contains` override to `BorderControl`**

  In `WorldViewportGump.cs`, add inside the `BorderControl` class body (before the closing `}` at line ~599):
  ```csharp
  public override bool Contains(int x, int y)
  {
      if (x < 0 || x > Width || y < 0 || y > Height)
          return false;
      const int hitMargin = 8;
      return y < hitMargin || y > Height - hitMargin
          || x < hitMargin || x > Width - hitMargin;
  }
  ```
